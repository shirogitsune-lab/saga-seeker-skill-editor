# Saga & Seeker キャラクターシートエディター

Saga & SeekerのキャラクターシートHTMLを新規作成し、基本情報、ステータス、スキル、性格キーワード、思い出、画像を安全に編集するWindows向けGUIアプリケーションです。AI向けMarkdownへの書出しと、Markdownから既知項目だけを取り込む新規作成にも対応します。実行ファイル、リポジトリ、Pythonパッケージの名称は互換性のため`SagaSeekerSkillEditor`／`saga-seeker-skill-editor`のままです。

## ダウンロードと起動

1. [最新版のダウンロードページ](https://github.com/shirogitsune-lab/saga-seeker-skill-editor/releases/latest)を開きます。
2. 通常は、ファイル名の末尾が`windows-x64-onefile.exe`になっているものをダウンロードします。
3. ダウンロードしたEXEを起動します。インストール作業は不要です。
4. 「既存シートを開く」「新規作成」「Markdownから新規作成」のいずれかを選び、5つの編集タブを使用します。
5. 編集後は「別名で保存」を使い、原本とは別のHTMLファイルとして保存します。

配布EXEはコード署名されていないため、Windows SmartScreenが警告を表示する場合があります。ダウンロードしたファイルは、Releaseに添付された`SHA256SUMS.txt`で検証できます。

配布形式は次の2種類です。

- `onefile`: 単体のEXE。通常はこちらを使用します。
- `onedir`: EXEと実行に必要なファイルをまとめたZIP。

このアプリケーションはローカルファイルだけを処理し、ネットワーク通信を行いません。

## 画面と操作

PySide6による5タブ構成の画面です。

- 「基本情報」: キャラクター名、プロフィール7項目、キャラクター画像
- 「ステータス」: 筋力、耐久力、知力、精神力、素早さ、運
- 「スキル」: 既存のマスター・ディテール型スキル編集
- 「性格キーワード」: ゲーム内カタログから最大6件を選択
- 「思い出」: 通常思い出と空白保持枠の追加、編集、削除、変換、並べ替え

互換用の「魅力」は画面に表示せず、既存値を保持します。新規シートでは`E`です。画像はPNG、JPEG、WebPを選択し、正方形切抜き後に512×512・WebP品質90で保存します。既存画像はプレビューを要求するまでデコードしません。

「基本情報」の通常編集では、プロフィール7項目の見出しを最初からすべて表示します。各見出しをクリックすると編集欄を個別に展開・折り畳みでき、複数項目を同時に開くこともできます。開閉しても入力内容とカーソル位置を保持し、開閉操作だけでは変更扱いになりません。プロフィール欄と画像欄の境界はドラッグで幅を調整できます。

プロフィール比較は初期状態では非表示です。必要なときに「プロフィール比較を表示」を押すと、7項目から異なる2項目を選び、同時に参照・編集できます。選択中の性格キーワードも比較領域へ表示されるため、「基本設定」やプロフィールの「性格」と整合を取りながら編集できます。左右の編集欄と性格キーワード参照の境界はドラッグで調整できます。「別ウィンドウで開く」を使うと比較領域だけを独立表示でき、ウィンドウまたは比較表示を閉じると再び非表示になります。どちらの表示も同じドラフトを編集し、表示・非表示を切り替えただけでは変更扱いになりません。

「性格キーワード」タブでは、ゲーム内に用意された150件から最大6件を選択できます。「力」「知恵」「富」「愛」「法」の系統と、「美徳」「中庸」「悪徳」の傾向を独立して絞り込めます。検索欄へ名前の一部を入力すると全系統から一致する候補を表示し、系統・傾向フィルターと組み合わせることもできます。候補を設定枠へドラッグして追加でき、設定済みキーワードはドラッグまたは「上へ移動」「下へ移動」で並び替えられます。ドラッグ中は有効な設定枠を枠線と文字で強調し、別の場所へドロップした場合は正しい操作先を表示します。途中のキーワードを削除すると後続項目を自動的に前へ詰めます。各項目のID、名前、系統、カルマをまとめて設定し、オリジナルの性格キーワードは作成しません。同じキーワードの重複や途中の未設定枠がある場合は、その場でエラーを表示して保存を止めます。

すべてのスキル枠を一覧表示し、未使用枠には複数のスキルをまとめて入力できます。途中の枠を空けたまま後ろの枠へ入力すると整合性エラーを表示し、手前から連続する状態へ修正されるまで保存を止めます。

スキルの削除やデフォルトスキルの置き換えは「高度な操作」に分離されています。リスト途中のスキルを削除する場合は、位置対応を維持するため空スキルへ置き換えます。末尾のスキルを削除する場合は、ゲームが自動追加に使用できる未使用枠へ戻します。

主なキーボードショートカット:

- `Ctrl+N`: 新しいキャラクターシートを作成する
- `Ctrl+O`: キャラクターシートHTMLを開く
- `Ctrl+Shift+S`: 編集結果を別名で保存する
- `F5`: 現在のファイルを再読み込みする
- `F6`: スキル一覧と編集フォームの間でフォーカスを移動する
- `Ctrl+F`: 性格キーワードの検索欄へフォーカスを移動する
- `Ctrl+Shift+P`: プロフィール比較を別ウィンドウで開く
- `Ctrl+Shift+M`: AI向けMarkdownを書き出す
- `Ctrl+Alt+M`: Markdownから新規作成する
- `Ctrl+W`: アプリケーションを終了する
- `Alt+↑` / `Alt+↓`: 選択中の思い出を並べ替える

未保存の変更がある状態でファイルの切り替え、再読み込み、終了を行う場合は、「別名で保存」「変更を破棄」「キャンセル」から処理を選択します。

「表示 > 外観」から、ライト、ダーク、ハイコントラストの3テーマを選択できます。初回起動時はライトテーマを使用し、次回以降は前回選択したテーマを復元します。「表示 > 起動時の表示」では、次回起動時にウィンドウ表示または最大化表示（全画面）のどちらを使用するか選択できます。

## AI向けMarkdown

「AI向けMarkdownを書き出す」は、現在の編集中ドラフトから次をUTF-8 Markdownへ出力します。

- キャラクター名、プロフィール7項目
- 性格キーワードと表示順
- 筋力、耐久力、知力、精神力、素早さ、運
- スキル名、説明、表示順（空の`type`を持つ有効なオリジナルスキルを含む）
- 通常思い出の全件と、概要・場所・意図・結果・タグ

画像、魅力、各種ID、内部のスキル種別・キー、空白保持枠、未使用スキル枠は出力しません。Markdown書出しはHTML保存ではないため、現在のHTML基準状態、保存先、未保存変更を更新しません。

「Markdownから新規作成」は完全な逆変換ではありません。既知の見出しから、キャラクター名、プロフィール7項目、性格キーワード、6ステータス、最大6件のスキルだけを読み込みます。画像は既定値、魅力は`E`、思い出は空、新しいIDと日時を使用します。Markdownにあったスキルは、元の内部ID・種別を推測せず、すべて新しいオリジナルスキルとして作成します。

Markdownからの新規作成は開始画面だけでなく、HTMLまたは新規シートを開いた後の画面上部からも実行できます。同じ場所にある「Markdownを書き出す」は現在のシートからAI向けMarkdownを生成する別操作です。

性格キーワードは同梱カタログとの完全一致が必要です。未知・重複・途中に空きのある枠、E～S以外のステータス、7件以上のスキル、既知見出しの重複は黙って補正せず、プレビューで停止します。HTMLやスクリプトは実行せず、UTF-8・8 MiB以下のMarkdownだけを固定見出しとして解析します。

## 安全仕様

- 入力HTML内のJavaScriptを実行しません。
- Markdown内のHTMLやJavaScriptも実行しません。解析・確認・生成が完了するまで現在のシートとドラフトを変更しません。
- ネットワーク通信を行いません。
- 未変更のシートを別名保存した場合は、入力と完全に同じバイトを出力します。編集値を元へ戻した場合も未変更へ戻ります。
- GUIへ表示しただけの値はドラフトへ書き戻しません。未編集のLF、CRLF、単独CRと既存JSONエスケープを保持します。
- 既知形式で一部セクションだけ不一致の場合は、そのセクションの読込時バイトと診断を固定し、正常セクションだけを編集できます。未知の`formatVersion`は全体読み取り専用です。
- キャラクター名は20文字、その他の入力項目は1000文字までを正常範囲とし、Python文字列のUnicodeコードポイント数で数えます。超過時はバグの可能性を警告しますが、確認後の保存は可能です。
- 思い出はJSON配列を正本とし、先頭6件だけをHTMLの直接`li`へ表示します。7件目以降もJSON内で保持・編集できます。
- 通常思い出と空白保持枠の合計が15件以上の場合、新規追加と補充だけを停止します。既存項目の編集、削除、並べ替え、種別変換は可能です。
- 思い出タグの順序、重複、前後空白、空文字、Unicode表現を自動変更しません。
- 既存思い出の空ID・重複IDと、空白保持枠の空IDを自動修復しません。
- 新規HTMLはCSPで外部通信を禁止し、JavaScript、イベント属性、外部リンク、外部画像、Webフォントを含みません。
- 画像はデコード前に64 MiB、最大辺16384 px、6400万画素、推定ピークメモリ256 MiBの上限を検査します。
- `data.skills`と`ul#skills-value > li`は、IDではなく配列位置と表示順で対応付けます。
- HTML側では`ul#skills-value`の直下にある`li`だけをスキル要素として扱います。
- 属性を持たない末尾の`<li>&nbsp;</li>`は未使用枠として扱い、対応する`data.skills`オブジェクトを要求しません。
- 明示的な空スキルオブジェクトは、枠を占有してゲームによる自動生成を防ぐ正式な`data.skills`要素です。未使用枠とは区別します。
- 余分なHTML要素に属性、子要素、表示文字がある場合は構造不一致として読み取り専用にします。
- 手動作成したスキルは、手前の未使用枠から連続するように追加します。連続する複数件を一度に保存できますが、途中に空欄がある場合は保存しません。
- リスト途中の削除は、対象のJSONオブジェクトと`li`だけを空スキルへ置き換えます。
- 末尾の削除は、最後のJSONオブジェクトだけを削除し、対応する`li`を未使用枠へ戻します。
- スキルIDの形式は制限しません。非文字列、空文字列、完全一致による重複だけを、対象スキル編集時の修復対象とします。
- 読み込み時から存在する空IDや重複IDは、自動修復しません。
- 保存時は同じフォルダに一時ファイルを作成し、`flush`、`fsync`、再読み込み検証を経てから`os.replace`で正式ファイルへ反映します。
- 利用者入力をJSONへ書き込む際は、`<`、`>`、`&`をUnicodeエスケープし、`</script>`によるscript要素の終了を防ぎます。
- 対象のJSONトークン、配列要素、対応HTML断片だけを置き換え、未知キー・未知属性と未変更断片を保持します。
- `data.personalities`と`ul#personality-value > li`も配列位置と表示順で対応付け、直下の6個の`li`だけを性格キーワード枠として扱います。
- 性格キーワードは同梱カタログの`id`、`name`、`type`、`karma`が完全一致する項目だけを編集対象とします。
- 未編集の性格キーワードオブジェクトと`li`はバイト単位で保持し、変更対象だけを置き換えます。

## 動作環境と依存関係

実行時依存:

- PySide6

開発・テスト・ビルド用依存:

- pytest
- PyInstaller
- Pillow

Pillowはビルド前のアイコン変換だけに使用します。アプリケーションの実行時には使用せず、配布EXEにも不要な依存として含めません。

## ソースからの実行

Python 3.11以降と`uv`を使用します。

```powershell
uv sync --extra dev --extra build
uv run python -m saga_seeker_skill_editor
```

テスト:

```powershell
uv run pytest -v
```

Windowsビルド:

```powershell
powershell -ExecutionPolicy Bypass -File .\build.ps1 -Mode onedir
powershell -ExecutionPolicy Bypass -File .\build.ps1 -Mode onefile
```

## ビルドについて

元のWebPアイコンは`assets/カナリア.webp`にあります。パッケージ作成前に`scripts/convert_icon.py`を実行すると、`assets/kanaria.ico`を生成します。

`SagaSeekerSkillEditor.spec`はEXEアイコンを設定し、onefile版へ`gui/styles/*.qss`、性格キーワードカタログ、新規シート用の既定WebP画像を同梱します。ビルドスクリプトはonedir版にも同じリソースを含めます。PyInstallerのonefile実行時に使用される`_MEIPASS`を含め、リソースパスの解決は一か所へ集約しています。

両形式の画像プラグイン回帰は次で確認できます。

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\exe_image_smoke.ps1 -ExePath .\dist\SagaSeekerSkillEditor\SagaSeekerSkillEditor.exe
powershell -ExecutionPolicy Bypass -File .\scripts\exe_image_smoke.ps1 -ExePath .\dist\SagaSeekerSkillEditor.exe
```

## テスト用HTML

GitHubへ公開するテストでは、最小限の合成フィクスチャだけを使用します。

実キャラクターシートHTML、Base64画像、個人設定、ゲーム由来コンテンツはGitへ追加しません。ローカル統合テスト用のHTMLは次の場所へ配置できます。

```text
tests/private_fixtures/
```

このディレクトリは`.gitkeep`以外がGitの追跡対象から除外されます。

別の非公開フォルダを使用する場合は、統合テスト実行前に`SAGA_SEEKER_PRIVATE_FIXTURES`へそのパスを設定します。設定したパス自体がリポジトリへ保存されることはありません。
